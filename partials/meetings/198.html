Java concurrency done with RxJava

There are couple of ways we can handle threads in Java, like using pure old Thread class, ExecutorService or CompletableFuture that emerged with Java 8. However, there is one another that seems to be more effective than those, which is also used by many known IT companies like Microsoft, Netflix or Github. It is a part of ReactiveX library and is called RxJava. This will be a live coding session presenting some of the RxJava's functionalities that we are using in one of our projects.
